RECOMMENDATIONS ENGINE LOGIC
For each enabled rule in rules_config.json:

Query ro.analytics.campaign_daily for recent data (match date window to condition metric: 7d or 30d)
Evaluate condition_metric against condition_operator + condition_value
If condition_2 exists, evaluate that too (AND logic)
If both conditions pass AND no existing pending recommendation exists for this campaign+rule combination:
Calculate proposed_value (current value ± action_magnitude %)
Generate human-readable trigger_summary
Assign confidence (use rule's risk_level as proxy: low=high confidence, medium=medium, high=low)
Insert new row into recommendations table with status = pending
Log how many recommendations were generated

roas_7d — 7-day ROAS vs target

One thing to flag before you sleep — cold email laws vary significantly by territory and you're targeting multiple countries:

UK — GDPR + PECR. B2B cold email is generally permitted to business email addresses with a legitimate interest basis, but needs an opt-out mechanism
USA — CAN-SPAM. More permissive for B2B, requires opt-out
Australia — Spam Act. Similar to UK, stricter than US
Canada — CASL. Strictest of all — requires implied or express consent
This doesn't make it unbuildable, it just means the system needs opt-out handling and suppression lists built in from day one — not bolted on later.
