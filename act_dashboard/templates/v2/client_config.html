Rule 7 exclusion tokens

Comma-separated tokens to EXCLUDE from Rule 7 (soft-signal blocking) even if they appear in your 1-word exact negative list. Use for industry-universal words (e.g. teeth, dental) that you neg as bare queries but don't want treated as block signals for multi-word terms. These tokens still work as exact-match negatives in Google Ads — this only prevents Rule 7 from extrapolating them.

How ACT Processes Search Terms

Every overnight, ACT runs three passes over each search term. Pass 1 classifies it. Pass 2 decides which negative list it would push to. Pass 3 looks for patterns across blocked terms and suggests consolidated phrase-match negatives to replace many individual exact-match negs.

Pass 1 — Classification (8 rules, first match wins)

#RuleWhat it checksConfig that affects itOutcome
1Brand ProtectionSearch term contains any brand phrase (substring match, order-sensitive)Brand Terms (Protect)Keep
2Existing Exact NegativeSearch term exactly equals a keyword in a 1–5+ WRDS exact-match negative listGoogle Ads negative lists (synced overnight)Block
3Existing Phrase NegativeA phrase-match negative appears as a contiguous word sequence in the search termGoogle Ads phrase-match negative lists (synced)Block
4Location Outside Service AreaAny location-shaped token (UK postcode outcode or city) is not in your service area. Postcode prefix-match falls back for short forms (sw1 → sw1a/sw1p/sw1v).Service LocationsBlock
5Services Offered - Not AdvertisedAll tokens of a not-advertised phrase appear in the search term (order-independent)Services Offered - Not Advertised + Auto-block toggleBlock (or Review if toggle off)
6Services Offered - AdvertisedAll tokens of an advertised phrase appear in the search term (order-independent)Services Offered - AdvertisedKeep
7Contains Negative VocabularyAny single query token matches a 1-word exact negative — excluding tokens in the Rule 7 exclusion listGoogle Ads 1 WRD [ex] list + Rule 7 exclusion tokens + Auto-block toggleBlock (or Review if toggle off)
8Ambiguous (no match)No rule above matched. Displays phrase-level near-match suggestions if any config phrase overlaps ≥50% on non-generic tokens.All config above (shows closest match for human decision)Needs Review

Pass 2 — Target List Routing (for Block and Review rows)

Determines which Google Ads negative list a search term would be pushed to if you approve it. Runs immediately after Pass 1, on every Block and Review row.

#Routing RuleConditionTarget list
1Denylist match overridePass 1 reason was service_not_advertised (Rule 5)Off Not Adv [ex]
2Word-count routing (default)Any other Block or Review reason — routed by word count of the search term1 WRD [ex] / 2 WRDS [ex] / 3 WRDS [ex] / 4 WRDS [ex] / 5+ WRDS [ex]

You can override the target list per row via the dropdown in the Search Term Review table.

Pass 3 — Phrase Suggestions (on demand)

Scans the day’s Block-classified search terms for shared sub-phrases. Proposes new phrase-match negatives that would catch multiple similar queries in one hit, reducing the list bloat of individual exact-match entries. Runs when you click Run Pass 3 on the Search Term Review page.

What it doesOutput
Finds contiguous word sequences (n-grams) that appear across multiple blocked search terms for the day. Ranks by how many terms each suggestion would consolidate. Suggestions appear in the Pass 3 Suggestions tab on Search Term Review, with the candidate phrase and a list of source search terms it would cover.
